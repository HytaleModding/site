O Kytale fornece vários utilitários que descomplicam o uso do Kotlin com o Hytale.

Enquanto aguardamos a liberação do JAR de servidor que tem a API do Hytale, temos diversos itens em nosso plano de desenvolvimento.

### Plano de Desenvolvimento para o Kytale

* DSLs (Linguagens Específicas de Domínio)
  * UI DSL — Vários protótipos estão em andamento, com base em exemplos de código de IU fornecidos por Ktar no X (antigo Twitter)
  * Desconhecido — Ainda estamos aguardando mais informações do Hytale. Adicionaremos mais conteúdos assim que possível
* Suporte kotlinx.coroutines — Com o lançamento do Hytale, avaliaremos se precisaremos de algo para melhorar o suporte às coroutines.
* Suporte kotlinx.serialization — (de)serializadores para os tipos do Hytale, a fim de garantir a integração com kotlinx.serialization.

O Kytale será publicado no Maven Central com as seguintes identificações:

* `io.github.hytalekt:kytale:version`
* `io.github.hytalekt:kytale-coroutines:version`
* `io.github.hytalekt:kytale-serialization:version`

Isso deve garantir que, se você não precisar de suporte para serialização/coroutines, não terá que incluí-lo.
