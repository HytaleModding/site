# Introdução

Um `Entity Component System`, também conhecido como ECS, é um padrão de design utilizado no desenvolvimento de jogos.

O ECS é composto por entidades, que referenciam componentes e por sistemas que operam sobre essas entidades.

Ele também enfatiza composição em vez de herança, eliminando hierarquias rígidas, ajudando a separar comportamento e dados, além de facilitar a reutilização.

# Composição sobre herança

Antes de entender a composição de ECS, vamos compreender a filosofia por trás dele.

Podemos descrever a composição como "tem X" e a herança como "é X".
Quando herdamos uma classe em Java, herdamos todos os seus atributos e métodos.
Na composição, podemos escolher o que ele referencia e quais sistemas atuarão sobre ele para cada objeto.

# Entidades, componentes e sistemas

## Entidades

Uma entidade é apenas um identificador único.

- Ela não contém dados;
- Ela não contém lógica.

Pense nas entidades como substantivos que podem ser descritos usando componentes.

## Componentes

Componentes são apenas contêineres simples de dados.

- Eles armazenam apenas dados;
- Eles não contêm comportamento nem lógica;
- Eles descrevem características ou aspectos de uma entidade.

Exemplos de componentes:

- Um componente "position" que armazena coordenadas X, Y e Z;
- Um componente "velocity" que armazena a velocidade ao longo de cada eixo;
- Um componente "health" que armazena a vida atual da entidade.

Pense nos componentes como adjetivos que descrevem uma entidade.

## Sistemas

Os sistemas contêm a lógica do seu jogo (ou, neste caso, de um mod).

- Operam sobre todas as entidades que têm um conjunto específico de componentes;
- Não armazenam dados das entidades por conta própria;
- Aplicam comportamento lendo e modificando os dados dos componentes.

Por exemplo:

- Um sistema "WinConditionSystem" pode processar toda entidade que contenha os componentes "Position" e "Velocity";
- Se uma entidade alcançar a posição `(0, 0, 0)`, o sistema acionará uma mensagem: "Você venceu!".
