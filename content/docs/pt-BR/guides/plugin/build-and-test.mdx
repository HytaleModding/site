## Criando seu mod

In our case, we've used **Gradle** as the build system. O Gradle é uma poderosa ferramenta de automação de build que gerencia dependências, compila seu código e empacota tudo em um arquivo JAR distribuível.

### Entendendo o Gradle

Gradle can use a `build.gradle.kts` file (Kotlin DSL) and `gradle.properties` to define your project's configuration, dependencies, and build settings. Você pode aprender mais sobre o Gradle no [site oficial do Gradle](https://gradle.org/).

### Construindo com Gradle

Para criar seu mod, abra um terminal no diretório do seu projeto e execute:

```bash
// Windows
gradlew.bat build

// Linux/MacOS
./gradlew build
```

Esse comando irá:

1. Compilar seu código-fonte Java
2. Executar testes (se existirem)
3. Empacotar tudo em um arquivo JAR

<Callout type="info">
Se você estiver usando o modelo de plugin do guia [Configurando seu ambiente de desenvolvimento](./setting-up-env), então o script de compilação já está configurado para você. It automatically grabs Hytale server jar/assets and handles the packaging of your mod, managing hytale dependencies and manifest changes.

You can change the project to using your game installs Asset zip by uncommenting a property to your `gradle.properties` file:
```properties
hytaleHomeOverride=/path/to/Hytale/install/release/package/game/latest/Assets.zip
```
and `build.gradle.kts` file:
```kts
hytaleHomeOverride = property("hytaleHomeOverride").toString()
```

</Callout>

### Saída da build

Quando a build for concluída com sucesso, seu mod estará localizado no diretório `build/libs` e será nomeado:

```
ExamplePlugin-1.0.jar
```

<Callout type="info">
The exact name may vary depending on what you configured in your `gradle.properties` file. Check the `mod_name` and `version` fields to see what your JAR will be named.
</Callout>

## Testando seu mod

Agora que você construiu seu mod, é hora de testá-lo no Hytale!

### 1. Localize a pasta de mods

Primeiro, você precisa encontrar o diretório de mods do Hytale:

1. Vá até o diretório de instalação do Hytale (o local padrão geralmente fica na pasta AppData)
   > No Windows, você pode acessá-lo clicando em `Windows + R` e digitando `%appdata%`.
2. Navegue até: `/UserData/Mods`

O caminho padrão deve se parecer com:

```
C:\Users\SeuUsuario\AppData\Roaming\Hytale\UserData\Mods
```

<Callout type="warning">
Se a pasta `Mods` não existir, talvez seja necessário criá-la manualmente ou iniciar o Hytale pelo menos uma vez.
</Callout>

### 2. Copie seu mod

Copie o arquivo `ExamplePlugin-1.0.jar` do diretório `build/libs` do seu projeto para a pasta `Mods` que você acabou de localizar.

### 3. Inicie o Hytale e verifique

1. **Inicie o Hytale**
2. **Clique em "Criar um Novo Mundo"**
3. **Clique no ícone de engrenagem de configurações**
4. **Clique em "Mods"**
5. **Veja o seu mod** na lista!

Se o seu mod aparecer na lista, parabéns!  Seu mod foi carregado com sucesso pelo Hytale.

## Testing Your Mod Without Building

Now that you have verified your project builds and runs in Hytale, you'll want to test it as you do development. To do this, simply run the `runServer` task or manually via:

```bash
// Windows
gradlew.bat runServer

// Linux/MacOS
./gradlew runServer
```

This will start and run a local Hytale server. You'll connect in your game adding a private server pointing to `localhost`. This server will run your mod and any dependencies you have added via your `build.gradle.kts` file.

## Solução de problemas

Se o seu mod não aparecer na lista de mods:

- Certifique-se de que o arquivo JAR está na pasta `Mods` correta
- Confirme se o arquivo `manifest.json` está configurado corretamente
- Procure por mensagens de erro nos logs do Hytale
- Verifique se você está usando a versão correta do Java e do Gradle
