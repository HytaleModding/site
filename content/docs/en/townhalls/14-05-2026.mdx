### Question (by haro0): Thoughts on the current userbase of Hytale?

### Question (by riprod): Are there any plans to improve Asset Loading or overriding other assets?
<Answer profile={KaupenjoeProfile}>
The note I have on that is, as they are sort of almost every major system, they're also reworking the asset system, and I hope that something like that would come in there, but that's as much detail as I have on that one.  On overriding assets, it works to some degree, but only with very specific things.  
</Answer>

<Answer profile={ZeroErrorsProfile}>
So basically the way that it works is that when you have a parent field that is set to “super”, it will automatically have the parent object of that asset use whatever is already loaded in the assets. So, for example if you have multiple packs and they load in a specific order, so you have the base game with a certain item, and you have your pack that gets loaded later, and it has the parent set to super, the pre-existing one that was loaded by the base-game will be the object that is the parent, and the child one when it loads will use that to do inheritance, and what happens is as it goes through and loads the fields in the CODEC, if the field is added with the inherited method, they have a 3rd argument that allows it to take a value from the parent object and add it to the current one. That's how the parenting system works.
</Answer>

### Question (by alechilles): When will we get Player Input, or the ability to read Player Input Directly?
<Answer profile={NeilProfile}>
I think there's a pretty clear precedent set right now that it'll come but it won't allow you read the keystrokes, but it would be it's own input system. Refer to the UnrealEngine's input system for an idea of what I'm talking about.
</Answer>

<Answer profile={ZeroErrorsProfile}>
We could like throw something out there, and see what happens, but we also don't want to risk potential other issues. Like we can't just give access to normal keybinds with all keys, because we want to make sure that there's a normal set that people are playing with, aswell as, we need to look down the road on how people are actually taking care of supporting that kind of thing for example (As we already have Controller support). There's a ton of things that play into it, and it becomes a larger topic than you think.
</Answer>

### Question (by haro0): What kind of testing tools or libraries are currently used in the development of Hytale?
<Answer profile={ZeroErrorsProfile}>
A lot of the stuff we have is in-house, at the base-level, it's just C# and Java. We have some tools that we built for automated testing, we do have the usual testing frameworks aswell. I mentioned in the Discord at one point that we also have a build tool, that helps build the game. That has built into it a test framework that boots up the client and a server and test some scenarios to make sure things are the way they are supposed to be. It's pretty limited right now, but we want to make sure that when we're picking up problems we can regression test and avoid them coming up again the future.
</Answer>

### Question (by quentin_qptech): The Instances system currently is not the best, do you think there would be improvements that you want to do that? Follow up to that, will there be ever a feature where you can add multiple entry and exit points to Instances.
<Answer profile={ZeroErrorsProfile}>
There are multiple levels here, at the base levels you have the server itself which supports multiple worlds, and on top of that you have instances, which allows you to have a asset that defines a world that has either a savegame or a pre-made world or a config for world generation. On top of that, there is the actual Instances in the game which you see with the Portals. It depends on what level you're talking about, on the gameplay side there will always be iteration to make sure the game is fun and enjoyable, but on the lower-level systems, if there is feedback on how Instances themselves or worlds within the server actually function, would love to hear.
</Answer>

### Question (by edilan): Is it possible or planned to be able to add new phobia modes?
<Answer profile={ZeroErrorsProfile}>
Potentially, I'm not sure at the moment. I believe there were a handful of things done there for very specific scenarios, but I can ask and see if there's any thoughts there. If you have anything specific you're thinking about, a lot of the problems with Phobia type systems is that there's very specific scenarios of things you need to override or change, and until we have some generic system that can do that a lot better, it's harder to do.
