# Contribuir a HytaleModding

Te damos la bienvenida a las contribuciones de la comunidad para ayudar a mejorar y ampliar la documentación de HytaleModding. Ya sea que estés corrigiendo errores tipográficos, añadiendo nuevos tutoriales o proporcionando ejemplos de código, ¡tus aportaciones son valiosas!

## Cómo contribuir

### Reportar problemas

Si has encontrado un problema en la documentación, notifícalo abriendo una nueva incidencia en nuestro [Repositorio GitHub] (https://github.com/HytaleModding/site/issues).

Si puedes trabajar en este problema por tu cuenta, puedes pedir que te lo asignen y enviar un PR.

### Envío de solicitudes de incorporación de cambios

1. Crea una bifurcación del repositorio en GitHub.
2. Clona tu rama en tu máquina local.
3. Crea una nueva rama para tus cambios.

```bash
git checkout -b my-feature-branch
```

4. Haz tus cambios y confírmalos con mensajes claros. Seguimos los [commits convencionales](https://www.conventionalcommits.org/es/v1.0.0/), Tu PR no será aceptada si los mensajes de tus commits no siguen esta convención.

```bash
git commit -m "feat: add feature X to documentation"
```

5. Aplicar los cambios a tu rama

```bash
git push origin my-feature-branch
```

6. Abre un Pull request en el repositorio original, describe los cambios y el porqué debería fusionarse.
