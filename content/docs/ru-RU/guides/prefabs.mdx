# Введение

Префабы в Hytale нужны для того, чтобы сохранять постройки и использовать их снова.

Вот что важно знать о префабах:

- С префабами работают в отдельном мире для редактирования.
- Сам префаб - это постройка в игровом мире, которая сохраняется в JSON-файл.

## Как создать префаб

Если что-то пойдёт не так, загляните в раздел "Известные проблемы" в конце этой статьи.

1. Используйте `/editprefab new <world name>`, чтобы создать новый мир для редактирования префабов: `/editprefab new my_prefab_world`
   - Обратите внимание, что `my_prefab` - это имя _мира_, а не самого префаба. В одном мире может быть несколько префабов.
2. Постройте в этом мире всё, что нужно.
3. Используйте кисть выделения, чтобы выбрать область, которую нужно сохранить как префаб.
4. Используйте `/prefab save`, чтобы сохранить постройку как префаб.
5. Используйте `/editprefab exit`, чтобы выйти из мира редактирования.
6. Возьмите Paste brush и нажмите `e` на клавиатуре, чтобы выбрать только что созданный префаб. Он будет в выпадающем списке `"server"` в правом верхнем углу меню.
   - Также его можно выбрать через `/prefab list`, найдя в списке только что созданный префаб.

С префабами можно делать намного больше, но выше показан базовый порядок действий. Ниже - дополнительная информация.

## Команды

Для префабов используются две основные команды: `/prefab` и `/editprefab`.

В этом списке показаны не все параметры команд. Чтобы увидеть доступные опции, добавьте флаг `--help` в конце команды, например `/prefab save --help`, или используйте `/help` в игре и найдите команды `prefab` и `editprefab`.

### `/prefab`

Эта команда нужна для работы с уже существующим префабом.

Подкоманды:

- `save`
  - Сохраняет префаб в файлы игры.
- `load`
  - Загружает префаб в игру.
- `delete`
  - Удаляет префаб из файлов игры.
- `list`
  - Показывает список всех сохранённых префабов.

### `/editprefab`

Команда `/editprefab` нужна, чтобы создать новый префаб или изменить уже существующий прямо в игре.

Подкоманды:

- `exit`
  - Выходит из текущего мира редактирования.
- `load`
  - Создаёт новый мир редактирования, куда можно вставить существующий префаб.
- `new`
  - Создаёт новый мир редактирования для создания префаба с нуля.
- `select`
  - Выделяет область префаба, на которую смотрит игрок, в пределах 200 блоков.
- `save`
  - Сохраняет текущий префаб, используя уже заданную область или текущую выделенную область.
- `saveui`
  - Открывает окно сохранения для текущего мира, где можно работать со всеми префабами в этом мире.
- `kill`
  - Удаляет все сущности в выбранном префабе.
- `saveas`
  - Сохраняет выбранный префаб в новый файл.
- `setbox`
  - Задаёт границы для выбранного префаба.
- `info`
  - Показывает общую информацию о выбранном префабе.
- `tp`
  - Открывает окно телепортации к префабу в текущем мире редактирования.
- `modified`
  - Показывает список всех изменённых префабов с несохранёнными изменениями в текущем мире редактирования.

# Известные проблемы

- После сохранения изменений в существующем префабе эти изменения могут не отображаться в игре.
  - Обходной путь: выйдите из мира и зайдите снова.
- При вставке префаба он не всегда отображается корректно.
  - Обходной путь: нажмите `t` на клавиатуре, чтобы переключить отображаемый материал; обычно хотя бы один режим отображается правильно.
- Команда `/prefab delete` может выдавать ошибку `Assert not in thread`.
