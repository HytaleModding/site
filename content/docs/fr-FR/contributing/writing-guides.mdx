## Créer un fichier de guide

Pour commencer à rédiger un guide, vous devez créer un fichier avec l'extension `.mdx`. Vous devez créer ce fichier dans le répertoire `/content/docs/en`, ensuite cela dépend de la section dans laquelle vous voulez faire le guide, vous pouvez donc décider dans quel dossier le placer après cela.

Supposons que je veuille faire un guide sur les bases de Java, je le placerais dans `/content/docs/en/guides/java-basics`. Vous devrez fouiller un peu pour trouver où vous voulez le placer.

<Callout>
  S'il existe déjà un guide dans la section sur laquelle vous voulez écrire, vous pouvez simplement cliquer sur le bouton « Modifier sur GitHub » sur notre site web et cela vous amènera au dossier, maintenant vous pouvez écrire dans le même dossier pour que votre guide apparaisse dans la même section.
</Callout>

## Frontmatter

Le frontmatter est les métadonnées de votre guide, il indique à notre site web comment afficher votre guide. Ceci doit être tout en haut du fichier, commençant par `---` et finissant par `---`.

Voici un exemple :

```mdx
---
title: "Les bases de Java"
description: "Apprenez les bases de la programmation Java."
authors:
    - name: "Votre Nom"
      url: "https://votresiteweb.com"
---
```

Comme vous pouvez le voir, c'est assez explicite, vous devez juste remplir les détails pour votre guide spécifique. Si vous n'avez pas d'URL vers laquelle vous voulez vous lier, ne mettez simplement pas cette ligne entière. De plus, la section auteurs est optionnelle, donc si vous ne voulez pas l'ajouter, vous pouvez simplement la laisser de côté bien que nous vous encouragions à le faire et à vous créditer.

L'indentation dans le frontmatter est importante, donc assurez-vous de la garder cohérente avec l'exemple.

## Structure du contenu

Vous devez structurer le contenu de votre guide de manière claire et organisée. Voici quelques conseils sur comment faire :

- **Introduction :** Commencez par une vue d'ensemble du sujet et ce que le lecteur peut s'attendre à apprendre.
- **Prérequis :** Listez tous les prérequis ou connaissances préalables que le lecteur devrait avoir avant de se plonger dans le guide.
- **Instructions étape par étape :** Décomposez le contenu en étapes claires et réalisables. Utilisez des titres et sous-titres pour organiser l'information.
- **Exemples de code :** Incluez des extraits de code pour illustrer les points clés. Assurez-vous d'expliquer chaque exemple minutieusement.
- **Conclusion :** Résumez les points principaux et encouragez le lecteur à explorer davantage ou à essayer ce qu'il a appris.

Maintenant, si vous écrivez une très grande section, vous devriez la diviser en plusieurs pages pour que ce soit plus facile à digérer pour le lecteur. Personne ne veut lire un gros bloc de texte d'un seul coup !

Lorsque nous écrivons des guides, nous devons aussi penser à comment les gens vont les consommer, et beaucoup de gens préfèrent survoler le contenu plutôt que de le lire en détail. C'est pourquoi il est préférable de diviser l'information en plusieurs pages, ainsi cela ressemble à une plus petite quantité d'information pour le lecteur. Aussi, pensez à utiliser des listes à puces, des listes numérotées et d'autres techniques de mise en forme pour rendre le contenu plus facile à parcourir.

## Mise en forme

Nous utilisons le [Markdown](https://www.markdownguide.org/) pour la mise en forme de nos guides. Voici quelques points clés à garder à l'esprit :

- Utilisez des titres (`#`, `##`, `###`) pour organiser votre contenu.
- Utilisez des listes à puces et des listes numérotées pour la clarté.
- Utilisez des blocs de code (\`\`\`) pour les extraits de code.
- Utilisez des liens pour référencer d'autres guides ou ressources externes.

C'est le même type de mise en forme que vous verrez sur la plupart des sites de documentation, ainsi que sur les plateformes de médias sociaux comme Discord.

Vous pouvez aussi faire un Callout pour mettre en évidence des informations importantes ou des astuces. Utilisez la syntaxe suivante :

```mdx
<Callout>
    Ceci est un message callout.
</Callout>
```

<Callout>
  Ceci est un message callout.
</Callout>

Vous pouvez changer la couleur du callout en ajoutant un attribut `type`. Par exemple :

```mdx
<Callout type="info">
    Ceci est un callout d'information.
</Callout>
```

Les types disponibles sont : `info`, `warning`, `error` et `success`.

## Conclusion

C'est tout ! Vous êtes maintenant un maître dans la rédaction de guides. Rappelez-vous simplement que vous n'avez pas besoin d'être parfait avec votre anglais, écrivez le guide comme vous expliqueriez le sujet à quelqu'un, et n'hésitez pas à demander des retours ou de l'aide si vous en avez besoin.

Bonne rédaction !
