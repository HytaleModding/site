# Directives pour les Pull Requests

Pour garantir un processus de relecture fluide et efficace, les contributeurs et les relecteurs doivent respecter les directives suivantes lors de la soumission et de la relecture des pull requests (PR).

## Pour les contributeurs

1. **Titres et descriptions explicites**
   - Utilisez des titres clairs et concis qui résument les modifications.
   - Fournissez des descriptions détaillées de ce que fait la PR, pourquoi les modifications sont nécessaires, et tout contexte pertinent.
2. **Lien vers les issues**
   - Dans le titre de votre PR, ajoutez `GH-NUMÉRO-ISSUE` pour lier la PR à l'issue concernée. Ceci est obligatoire pour toutes les PR.
3. **Modifications petites et ciblées**
   - Visez à garder les PR petites et concentrées sur une seule issue ou fonctionnalité. Cela facilite la compréhension et les retours des relecteurs.
4. **Qualité du code**
   - Assurez-vous que votre code respecte les normes et bonnes pratiques de codage du projet.
   - Incluez des commentaires si nécessaire pour expliquer une logique complexe.
5. **Tests**
   - Testez vos modifications minutieusement avant de soumettre la PR.
   - Le cas échéant, incluez des tests unitaires ou d'intégration pour vérifier vos modifications.
6. **Documentation**
   - Mettez à jour ou ajoutez de la documentation si nécessaire pour refléter vos modifications.
7. **Messages de commit**
   - Utilisez des messages de commit clairs et descriptifs qui suivent le format [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/).
8. **Vérifier les conflits**
   - Avant de soumettre votre PR, assurez-vous que votre branche est à jour avec la branche de base pour minimiser les conflits de fusion.

## Pour les relecteurs

- Vous êtes censé relire les PR minutieusement et demander des modifications si nécessaire.
- Fournissez des retours constructifs et des suggestions d'amélioration.
- Vérifiez que les modifications respectent les normes et bonnes pratiques de codage du projet.
- Déployez la PR en utilisant l'intégration dokploy pour vous assurer que les modifications fonctionnent comme prévu.
- Assurez-vous que la documentation est mise à jour si nécessaire.
- N'approuvez la PR que lorsque vous êtes satisfait des modifications et que toutes les vérifications sont passées.
