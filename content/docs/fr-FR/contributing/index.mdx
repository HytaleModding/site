# Contribuer à HytaleModding

Nous accueillons les contributions de la communauté pour nous aider à améliorer et enrichir la documentation de HytaleModding. Que vous corrigiez des fautes de frappe, ajoutiez de nouveaux tutoriels ou fournissiez des exemples de code, vos contributions sont précieuses !

## Comment contribuer

### Signaler des problèmes

Si vous avez trouvé un problème dans la documentation, veuillez le signaler en ouvrant une nouvelle issue sur notre [dépôt GitHub](https://github.com/HytaleModding/site/issues).

Si vous pouvez travailler sur ce problème vous-même, vous pouvez demander à y être assigné et soumettre une PR.

### Soumettre des Pull Requests

1. Forkez le dépôt sur GitHub.
2. Clonez votre fork sur votre machine locale.
3. Créez une nouvelle branche pour vos modifications.

```bash
git checkout -b my-feature-branch
```

4. Effectuez vos modifications et validez-les avec des messages clairs. Nous suivons les [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/), votre PR ne sera pas acceptée si vos messages de commit ne suivent pas cette convention.

```bash
git commit -m "Add feature X to documentation"
```

5. Poussez vos modifications vers votre fork.

```bash
git push origin my-feature-branch
```

6. Ouvrez une Pull Request sur le dépôt original, en décrivant vos modifications et pourquoi elles devraient être fusionnées.
