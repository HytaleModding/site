Les capacités de modding de Hytale incluent un système de modding basé sur Java qui permet aux moddeurs de créer des comportements personnalisés, des interactions et des mécaniques de jeu. Bien que les détails complets de ce système ne soient pas documentés publiquement car le jeu est encore en développement et non publié, nous pouvons résumer ce qui est actuellement connu à son sujet.

### Détails importants du modding dans Hytale

Le modding de Hytale se fait côté serveur et non côté client, ce qui signifie que les mods/plugins sont hébergés sur le serveur et les joueurs se connectent à ces serveurs moddés pour vivre le contenu personnalisé sans avoir besoin d'installer quoi que ce soit de leur côté. Cette approche garantit une expérience cohérente pour tous les joueurs sur un serveur moddé.

### Aperçu du modding basé sur Java

Le système de modding côté serveur dans Hytale est basé sur Java, ce qui fait des plugins serveur des fichiers JAR.

Nous supposons que cela a été fait pour que les moddeurs Minecraft puissent facilement faire la transition vers le modding Hytale, puisque le modding Minecraft est également principalement basé sur Java. Cela permet aux moddeurs familiers avec Java d'exploiter leurs connaissances existantes lors de la création de mods pour Hytale.

#### Préoccupations pour le solo

Malgré le nom, les plugins serveur peuvent aussi être utilisés en mode solo. Le mode solo de Hytale exécute une instance de serveur locale, permettant aux joueurs d'utiliser des plugins serveur même lorsqu'ils jouent seuls. Cela signifie que les mods créés comme plugins serveur peuvent également améliorer l'expérience solo.

### Système Mixin

Les Mixins étaient utilisés dans le modding Minecraft pour modifier directement le code du jeu, pouvant même écraser ou rediriger la logique et les valeurs. Un système extrêmement puissant mais quelque peu fragile qui dépend du fait que le code source/version du jeu reste à peu près le même.

Nous n'avons pas d'information officielle sur les Mixins dans Hytale. Malgré cela, nous avons des raisons de croire qu'ils n'auront pas besoin d'être supportés pour les raisons suivantes :

1. Pour un jeu correctement conçu, 95% des cas d'utilisation de Mixin peuvent être résolus avec une bonne API à la place. En fait, de nombreuses APIs de modding Minecraft font simplement du mixin pour fournir l'API afin que les développeurs utilisant ces APIs n'aient pas à écrire le mixin eux-mêmes.
   * Il s'agit généralement de remplacer des valeurs codées en dur et d'écraser des comportements codés en dur.
   * Hytale devrait idéalement rendre ces deux éléments soit API soit pilotés par les données.
2. Les APIs peuvent aussi être beaucoup plus résilientes que la version du jeu. Nous pouvons avoir un système de version d'API séparé, ce qui signifie que le même code n'aura pas besoin d'être modifié à travers plusieurs mises à jour jusqu'à ce que l'API change de manière cassante.
