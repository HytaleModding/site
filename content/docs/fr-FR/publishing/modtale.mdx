# Publier Votre Mod sur Modtale

[Modtale](https://modtale.net) est le dépôt communautaire Hytale pour découvrir des mods, asset packs, worlds et modpacks.

Ce guide vous accompagnera à travers le processus complet de publication de votre mod.

## Prérequis

Avant de commencer, assurez-vous d'avoir :

- Un fichier `.jar` compilé de votre plugin, ou `.zip` de votre asset pack
- Une description des fonctionnalités de votre mod

Nous recommandons également :

- Des captures d'écran ou images de haute qualité de votre mod en action
- Un logo ou icône clair (taille recommandée : 512x512px)
- Toutes les dépendances ou exigences documentées

## Processus de Publication Étape par Étape

### 1. Configuration du Compte

Connectez-vous à [Modtale](https://modtale.net) en utilisant votre méthode préférée. Si vous n'avez pas de compte, créez-en un d'abord.

### 2. Commencer la Création

Cliquez sur le bouton **"Create"** dans la barre de navigation ou allez directement sur la [page de téléchargement](https://modtale.net/upload).

### 3. Choisir le Type de Contenu

Sélectionnez le type de contenu approprié pour votre soumission :

- **Server Plugins** - Pour les plugins et modifications côté serveur Hytale
- **Data Assets** - Pour les packs de données personnalisés, tables de butin et autre contenu piloté par les données
- **Art Assets** - Pour les packs de textures, modèles et autre contenu artistique
- **Worlds** - Pour les mondes Hytale personnalisés
- **Modpacks** - Pour les collections organisées de mods et assets

![Content Type Selection](/assets/guides/modtale_type.png)

### 4. Informations de Base

Remplissez les détails essentiels :

- **Name**: Choisissez un nom clair et descriptif pour votre mod
- **Summary**: Rédigez une brève description (1-2 phrases, 250 caractères max) qui explique ce que fait votre mod

![Project Title and Description](/assets/guides/modtale_title.png)

### 5. Configuration du Projet

Vous êtes maintenant sur la page de gestion de votre projet. Ici vous pouvez :

#### Télécharger des Fichiers

Naviguez vers l'onglet **"Files"** et téléchargez votre fichier `.jar` ou `.zip` :

- Glissez-déposez votre fichier ou utilisez le bouton de téléchargement
- Ajoutez des notes de version décrivant les changements et nouvelles fonctionnalités
- Marquez le type de version (Release, Beta, Alpha)

![File Upload Interface](/assets/guides/modtale_upload.png)

#### Personnaliser Votre Projet

- **Logo**: Téléchargez un logo de haute qualité (512x512 recommandé)
- **Banner**: Téléchargez une bannière pour votre page (1920x640 recommandé)
- **Screenshots**: Ajoutez plusieurs captures d'écran montrant votre mod en action
- **Description**: Rédigez une description complète en utilisant le formatage Markdown
- **Links**: Ajoutez des liens vers votre code source, serveur Discord ou documentation
- **Tags**: Ajoutez des tags pertinents pour aider les utilisateurs à trouver votre mod
- **License**: Indiquez quelle licence votre mod utilise parmi une sélection de licences courantes, ou utilisez une licence personnalisée

## Meilleures Pratiques

### Documentation

<Callout type="info">
  Incluez toujours une documentation complète dans la description de votre mod. Les utilisateurs doivent comprendre ce que fait votre mod, comment l'utiliser et toutes les exigences spéciales.
</Callout>

### Notes de Version

Lors du téléchargement de nouvelles versions :

- Vous devez respecter SemVer
- Documentez clairement tous les changements et nouvelles fonctionnalités
- Incluez les corrections de bugs et problèmes connus
- Mentionnez tout changement cassant ou étapes de migration
- Listez les nouvelles dépendances ou changements d'exigences

### Attrait Visuel

- Utilisez des captures d'écran de haute qualité qui mettent en valeur les fonctionnalités de votre mod dans la galerie
- Créez un logo accrocheur qui représente votre mod
- Envisagez d'ajouter une image de bannière pour votre page de projet

### Engagement Communautaire

- Répondez rapidement aux commentaires et retours
- Envisagez de créer un serveur Discord pour le support communautaire
- Mettez régulièrement à jour votre mod en fonction des retours des utilisateurs

## Après la Publication

### Mises à jour automatisées

- Vous pouvez configurer votre projet pour télécharger automatiquement de nouvelles versions en utilisant Github Actions, Gradle, Maven ou Go. Voir des exemples [ici](https://github.com/Modtale/modtale-example)

### Surveillance de Votre Projet

- Consultez les analyses pour voir les statistiques de téléchargement
- Surveillez les commentaires et avis
- Gardez une trace des rapports de bugs et demandes de fonctionnalités

### Marketing de Votre Mod

- Partagez votre mod sur les communautés Hytale pertinentes
- Créez des vidéos de présentation ou tutoriels
- Engagez-vous avec d'autres moddeurs et créateurs

<Callout type="warning">
  N'oubliez pas de maintenir votre mod à jour pour les nouvelles versions de Hytale et de maintenir la compatibilité avec le framework de modding de Hytale pour maximiser la portée de votre audience.
</Callout>

## Obtenir de l'Aide

Si vous rencontrez des problèmes pendant le processus de publication :

- Rejoignez leur communauté Discord pour obtenir du support
- Contactez directement l'équipe Modtale via leurs canaux de support
