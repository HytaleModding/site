Les méthodes sont des blocs de code réutilisables qui effectuent des tâches spécifiques. Elles vous aident à éviter de répéter du code et rendent vos programmes plus faciles à comprendre.

## Qu'est-ce qu'une Méthode ?

Pensez à une méthode comme une recette. Vous la définissez une fois, puis l'utilisez chaque fois que vous en avez besoin.

```java tab="Code"
public class Game {
    public static void main(String[] args) {
        greet();        // Call the method
        greet();        // Call it again!
    }
    
    // Method definition
    public static void greet() {
        System.out.println("Welcome to Hytale!");
    }
}
```

```text tab="Output"
Welcome to Hytale!
Welcome to Hytale!
```

## Structure d'une Méthode

```java
public static void methodName() {
//  ^      ^     ^      ^       ^
//  |      |     |      |       |
// access static return name  parameters
// modifier       type
    // Code goes here
}
```

Nous apprendrons ce que signifient `public` et `static` plus tard. Pour l'instant, utilisez-les simplement.

## Méthodes avec Paramètres

Les paramètres vous permettent de passer des données dans les méthodes :

```java
public static void greetPlayer(String name) {
    System.out.println("Hello, " + name + "!");
}

public static void main(String[] args) {
    greetPlayer("Alice");  // Hello, Alice!
    greetPlayer("Bob");    // Hello, Bob!
}
```

### Paramètres Multiples

```java
public static void dealDamage(String target, int damage) {
    System.out.println(target + " takes " + damage + " damage!");
}

public static void main(String[] args) {
    dealDamage("Zombie", 20);    // Zombie takes 20 damage!
    dealDamage("Skeleton", 15);  // Skeleton takes 15 damage!
}
```

<Callout type="info" title="L'Ordre des Paramètres Compte">
  Lors de l'appel d'une méthode, les arguments doivent correspondre à l'ordre des paramètres :

  ```java
  public static void createItem(String name, int quantity, double price) {
      // ...
  }

  // Correct
  createItem("Sword", 1, 50.0);

  // Incorrect - L'ordre compte !
  createItem(1, "Sword", 50.0);  // Error!
  ```
</Callout>

## Méthodes avec Valeurs de Retour

Les méthodes peuvent renvoyer des données en utilisant `return` :

```java
public static int add(int a, int b) {
    int sum = a + b;
    return sum;
}

public static void main(String[] args) {
    int result = add(5, 3);
    System.out.println(result);  // 8
}
```

Le type de retour doit correspondre à ce que vous retournez :

* `void` - Ne retourne rien
* `int` - Retourne un entier
* `double` - Retourne un décimal
* `boolean` - Retourne vrai/faux
* `String` - Retourne du texte

```java
public static String getItemName() {
    return "Diamond Sword";
}

public static boolean isPlayerAlive(int health) {
    return health > 0;
}

public static double calculateDamage(int attack, double multiplier) {
    return attack * multiplier;
}
```

<Callout type="warning" title="Return Arrête l'Exécution">
  Une fois qu'une méthode atteint `return`, elle se termine immédiatement. Le code après `return` ne s'exécutera pas !

  ```java
  public static int getValue() {
      return 10;
      System.out.println("This never runs!");  // Unreachable code!
  }
  ```
</Callout>

## Exemples Pratiques

### Système de Santé

```java tab="Code"
public static void displayHealth(String name, int health, int maxHealth) {
    double percentage = (health * 100.0) / maxHealth;
    System.out.println(name + ": " + health + "/" + maxHealth + 
                      " (" + percentage + "%)");
}

public static void main(String[] args) {
    displayHealth("Player", 75, 100);
    displayHealth("Boss", 450, 500);
}
```

```text tab="Output"
Player: 75/100 (75.0%)
Boss: 450/500 (90.0%)
```

### Calculateur de Dégâts

```java tab="Code"
public static int calculateDamage(int baseAttack, int weaponDamage, boolean isCritical) {
    int totalDamage = baseAttack + weaponDamage;
    
    if (isCritical) {
        totalDamage *= 2;
    }
    
    return totalDamage;
}

public static void main(String[] args) {
    int damage1 = calculateDamage(10, 15, false);  // 25
    int damage2 = calculateDamage(10, 15, true);   // 50
    
    System.out.println("Normal hit: " + damage1);
    System.out.println("Critical hit: " + damage2);
}
```

### Exigences de Niveau

```java tab="Code"
public static int getXPForLevel(int level) {
    return level * 100;
}

public static boolean canLevelUp(int currentXP, int currentLevel) {
    int required = getXPForLevel(currentLevel + 1);
    return currentXP >= required;
}

public static void main(String[] args) {
    int playerXP = 450;
    int playerLevel = 4;
    
    if (canLevelUp(playerXP, playerLevel)) {
        System.out.println("You can level up!");
    } else {
        int needed = getXPForLevel(playerLevel + 1) - playerXP;
        System.out.println("Need " + needed + " more XP");
    }
}
```

### Calculateur de Distance

```java tab="Code"
public static double calculateDistance(int x1, int y1, int x2, int y2) {
    int dx = x2 - x1;
    int dy = y2 - y1;
    return Math.sqrt(dx * dx + dy * dy);
}

public static boolean isInRange(int x1, int y1, int x2, int y2, double range) {
    double distance = calculateDistance(x1, y1, x2, y2);
    return distance <= range;
}

public static void main(String[] args) {
    // Check if enemy is in attack range
    if (isInRange(0, 0, 5, 5, 10.0)) {
        System.out.println("Target in range!");
    }
}
```

## Surcharge de Méthode

Vous pouvez avoir plusieurs méthodes avec le même nom mais des paramètres différents :

```java
public static void displayMessage(String message) {
    System.out.println(message);
}

public static void displayMessage(String message, int times) {
    for (int i = 0; i < times; i++) {
        System.out.println(message);
    }
}

public static void main(String[] args) {
    displayMessage("Hello");           // Calls first version
    displayMessage("Hello", 3);        // Calls second version
}
```

<Callout type="info" title="Règles de Surcharge de Méthode">
  Les méthodes sont considérées comme différentes si elles ont :

  * Un nombre différent de paramètres
  * Des types de paramètres différents
  * Un ordre de paramètres différent

  ```java
  public static void test(int a) { }
  public static void test(int a, int b) { }        // Different count
  public static void test(double a) { }            // Different type
  public static void test(String a, int b) { }     // Different types
  public static void test(int a, String b) { }     // Different order

  // Incorrect - Seul le type de retour est différent
  public static int test(int a) { }
  ```
</Callout>

## Modèles Courants dans le Modding Hytale

### Création d'Objet

```java
public static Item createItem(String name, int durability) {
    Item item = new Item();
    item.setName(name);
    item.setDurability(durability);
    return item;
}

public static Item createSword() {
    return createItem("Sword", 100);
}

public static Item createPickaxe() {
    return createItem("Pickaxe", 150);
}
```

### Validation de Placement de Bloc

```java
public static boolean canPlaceBlock(int x, int y, int z) {
    // Check if position is valid
    if (y < 0 || y > 255) {
        return false;
    }
    
    // Check if block already exists
    if (isBlockAt(x, y, z)) {
        return false;
    }
    
    return true;
}

public static void placeBlock(int x, int y, int z, String type) {
    if (canPlaceBlock(x, y, z)) {
        // Place the block
        System.out.println("Placed " + type + " at (" + x + ", " + y + ", " + z + ")");
    } else {
        System.out.println("Cannot place block there!");
    }
}
```

### Vérifications d'État du Joueur

```java
public static boolean isLowHealth(int health, int maxHealth) {
    return (health * 100.0 / maxHealth) < 25;
}

public static String getHealthStatus(int health, int maxHealth) {
    double percentage = (health * 100.0) / maxHealth;
    
    if (percentage >= 75) {
        return "Healthy";
    } else if (percentage >= 50) {
        return "Injured";
    } else if (percentage >= 25) {
        return "Critical";
    } else {
        return "Near Death";
    }
}
```

## Meilleures Pratiques

<Callout type="info" title="Nommage des Méthodes">
  Utilisez des noms descriptifs basés sur des verbes qui expliquent ce que fait la méthode :

  ```java
  // Bon
  public static void calculateDamage() { }
  public static boolean isPlayerAlive() { }
  public static String getItemName() { }
  public static void displayInventory() { }

  // Mauvais
  public static void dmg() { }        // Too short
  public static void method1() { }    // Not descriptive
  public static void stuff() { }      // Too vague
  ```
</Callout>

## Exercices Pratiques

1. **Convertisseur de Température** : Écrivez des méthodes pour convertir :
   * Celsius vers Fahrenheit : `(C × 9/5) + 32`
   * Fahrenheit vers Celsius : `(F - 32) × 5/9`

2. **Calculateur de Cercle** : Créez des méthodes qui calculent :
   * Aire : `π × rayon²`
   * Circonférence : `2 × π × rayon`
   * Utilisez `Math.PI` pour π

3. **Durabilité d'Objet** : Écrivez ces méthodes :
   * `damageItem(int current, int damage)` - retourne la nouvelle durabilité
   * `isBroken(int durability)` - retourne vrai si la durabilité est inférieure ou égale à 0
   * `repairItem(int current, int max)` - retourne la durabilité maximale

4. **Validateur de Mot de Passe** : Créez une méthode qui vérifie si un mot de passe est valide :
   * Au moins 8 caractères
   * Contient au moins un chiffre
   * Retourne vrai si valide, faux sinon

## Erreurs Courantes

```java
// Incorrect - Oubli de l'instruction return
public static int getValue() {
    int x = 10;
    // Forgot to return!
} !

// Correct
public static int getValue() {
    int x = 10;
    return x;
}

// Incorrect - Mauvais type de retour
public static int getText() {
    return "Hello";  // Error! Should return int, not String
}

// Correct
public static String getText() {
    return "Hello";
}

// Incorrect - Ne pas appeler la méthode
public static void main(String[] args) {
    greet;  // Error! Missing ()
}

// Correct
public static void main(String[] args) {
    greet();
}
```
