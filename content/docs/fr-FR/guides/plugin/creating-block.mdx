La création de blocs personnalisés nécessite d'activer les packs de ressources et d'organiser vos fichiers correctement.

## Configuration

1. Activez les packs de ressources dans `manifest.json` en définissant `IncludesAssetPack` à `true`
2. Créez la structure de dossiers suivante :

<Files>
  <Folder name="resources" defaultOpen>
    <Folder name="Server" defaultOpen>
      <Folder name="Item" defaultOpen>
        <Folder name="Items" />
      </Folder>

      <Folder name="Languages" defaultOpen>
        <Folder name="en-US" />
      </Folder>
    </Folder>

    <Folder name="Common" defaultOpen>
      <Folder name="Icons" />

      <Folder name="Blocks" defaultOpen>
        <Folder name="my_new_block" defaultOpen>
          <File name="model.blockymodel" />
        </Folder>
      </Folder>

      <Folder name="BlockTextures" defaultOpen>
        <File name="texture.png" />
      </Folder>
    </Folder>
  </Folder>
</Files>

3. Ajoutez les traductions dans `Server/Languages/en-US/items.lang` :

```
my_new_block.name = My New Block
my_new_block.description = My Description
```

## Bloc Cube Basique

Créez `Server/Item/Items/my_new_block.json` :

```json
{
  "TranslationProperties": {
    "Name": "items.my_new_block.name",
    "Description": "items.my_new_block.description"
  },
  "Id": "My_New_Block",
  "MaxStack": 100,
  "Icon": "Icons/ItemsGenerated/my_new_block.png",
  "Categories": [
    "Blocks.Rocks"
  ],

  "PlayerAnimationsId": "Block",
  "Set": "Rock_Stone",
  "BlockType": {
    "Material": "Solid",
    "DrawType": "Cube",
    "Group": "Stone",
    "Flags": {},
    "Gathering": {
      "Breaking": {
        "GatherType": "Rocks",
        "ItemId": "my_new_block"
      }
    },
    "BlockParticleSetId": "Stone",
    "Textures": [
      {
        "All": "BlockTextures/my_new_block.png"
      }
    ],
    "ParticleColor": "#aeae8c",
    "BlockSoundSetId": "Stone",
    "BlockBreakingDecalId": "Breaking_Decals_Rock"
  },
  "ResourceTypes": [
    {
      "Id": "Rock"
    }
  ]
}
```

Assurez-vous que tous les fichiers référencés existent dans leurs dossiers respectifs, et que les chemins sont corrects.
