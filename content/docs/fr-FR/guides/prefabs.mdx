# Introduction

Les prefabs sont la façon pour Hytale de créer des structures réutilisables.

Il y a quelques concepts fondamentaux à comprendre sur les prefabs :

- Les utilisateurs interagissent avec les prefabs via un monde d'édition de prefabs.
- Les prefabs sont les structures physiques dans le monde de jeu, et sont sauvegardés comme fichiers JSON.

## Comment créer un Prefab

Assurez-vous de vérifier la section Problèmes connus à la fin de ce guide si vous rencontrez des problèmes.

1. `/editprefab new <nom du monde>` pour créer un nouveau monde d'édition de prefab : `/editprefab new my_prefab_world`
   - Notez que `my_prefab` est le nom du _monde_, pas du prefab lui-même. Les mondes peuvent contenir plusieurs prefabs.
2. Construisez ce que vous voulez dans le monde d'édition de prefab.
3. Utilisez le pinceau de sélection pour sélectionner la zone que vous voulez sauvegarder comme prefab.
4. `/prefab save` pour sauvegarder la structure comme prefab.
5. `/editprefab exit` pour quitter le monde d'édition de prefab.
6. En utilisant le pinceau de collage, appuyez sur 'e' sur votre clavier pour sélectionner le prefab que vous venez de créer. Il sera dans le menu déroulant « server », qui peut être sélectionné en haut à droite du menu.
   - Vous pouvez aussi le sélectionner avec `/prefab list` et en sélectionnant le prefab que vous venez de créer.

Il y a beaucoup plus que vous pouvez faire avec les prefabs ; ceci n'est que le flux de base. Continuez à lire pour plus d'informations.

## Commandes

Il y a deux commandes principales pour les prefabs, `/prefab` et `/editprefab`.

Cette liste ne couvre pas les différentes options pour chaque commande. Vous pouvez utiliser le flag `--help` à la fin de n'importe quelle commande pour voir les options disponibles, par ex. `/prefab save --help`, ou utiliser `/help` en jeu et trouver les commandes `prefab` et `editprefab`.

### `/prefab`

Cette commande permet à l'utilisateur d'interagir avec le fichier d'un prefab existant.

Sous-commandes :

- `save`
  - Sauvegarde le prefab dans le système de fichiers.
- `load`
  - Charge le prefab dans le jeu.
- `delete`
  - Supprime le prefab du système de fichiers.
- `list`
  - Liste tous les prefabs dans le système de fichiers.

### `/editprefab`

La commande `/editprefab` est utilisée pour créer ou éditer la structure physique dans le jeu que le prefab contient.

Sous-commandes :

- `exit`
  - Quitte le monde d'édition de prefab actuel.
- `load`
  - Crée un nouveau monde d'édition de prefab pour y coller un prefab existant.
- `new`
  - Crée un nouveau monde d'édition de prefab pour créer un nouveau prefab de zéro.
- `select`
  - Sélectionne la zone du prefab que l'utilisateur regarde (dans un rayon de 200 blocs).
- `save`
  - Sauvegarde le prefab actuel en utilisant la zone existante, ou la zone actuellement sélectionnée.
- `saveui`
  - Ouvre l'interface de sauvegarde pour le monde actuel, permettant à l'utilisateur d'interagir avec tous les prefabs qui sont dans le monde.
- `kill`
  - Fait disparaître toutes les entités dans le prefab actuellement sélectionné.
- `saveas`
  - Sauvegarde le prefab sélectionné dans un nouveau fichier.
- `setbox`
  - Définit la boîte englobante du prefab actuellement sélectionné.
- `info`
  - Informations générales sur le prefab actuellement sélectionné.
- `tp`
  - Ouvre l'interface de téléportation pour se téléporter vers un prefab dans le monde d'édition actuel.
- `modified`
  - Liste tous les prefabs modifiés avec des changements non sauvegardés dans le monde d'édition actuel.

# Problèmes connus

- Après avoir sauvegardé des modifications d'un prefab existant, le prefab ne reflète pas les changements dans le jeu.
  - Solution de contournement : quittez et réentrez dans le monde.
- Lors du collage d'un prefab, le prefab n'est pas toujours affiché avec précision.
  - Solution de contournement : appuyez sur 't' sur votre clavier pour basculer le matériau affiché ; généralement au moins une vue est exacte.
- `/prefab delete` génère une erreur avec `Assert not in thread`.
